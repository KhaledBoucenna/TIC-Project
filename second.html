  1. Introduction au Web et à Internet :
    2.

        Dans cette introduction, nous abordons la relation entre le Web et Internet. Le Web, ou World Wide Web (WWW), est un système hypertexte public sur Internet qui permet aux utilisateurs de consulter des pages web via un navigateur. Le Web est une application spécifique d'Internet, qui est un support de communication général.
    Principes du World Wide Web :
        Le World Wide Web repose sur le principe d'accéder à des documents textuels situés sur des machines accessibles par Internet. Ces documents sont reliés entre eux par des liens hypertexte, créant ainsi une "toile" d'informations interconnectées. Le Web sert à fournir diverses ressources, telles que du texte, des images, des sons et des vidéos, du contenu applicatif et interactif.
    URL (Uniform Resource Locator) :
        Une URL, ou Uniform Resource Locator, est un lien hypertexte permettant de suivre un lien et d'ouvrir une nouvelle page web. Elle contient l'adresse du serveur où se trouve la ressource et l'adresse spécifique de la page sur le serveur. Par exemple: http://example.com/un/chemin/page.html.

  2. Aspects techniques du Web :
    3.

        Dans cette section, nous abordons les aspects techniques fondamentaux du Web.
    I) URL (Uniform Resource Locator) :
        Les URL constituent l'un des mécanismes clés du Web. Elles permettent aux utilisateurs d'accéder à un ensemble de ressources en spécifiant l'emplacement des ressources sur des serveurs accessibles via Internet. Le mécanisme de localisation peut faire appel au protocole DNS pour traduire les noms de domaine en adresses IP.
    II) HTTP (Hypertext Transfer Protocol) :
        Le protocole HTTP (Hypertext Transfer Protocol) est un protocole de niveau applicatif utilisé pour les communications sur le Web. Il fonctionne selon un modèle client-serveur, où le navigateur agit en tant que client et les serveurs web répondent aux requêtes. HTTP est un protocole sans état, ce qui signifie qu'il ne conserve pas la mémoire des transactions précédentes.
    III) HTML (HyperText Markup Language) :
        HTML est un langage de description utilisé pour structurer et présenter les pages web. Il permet d'inclure du texte, des images et d'autres objets dans les pages. HTML prend également en charge les liens hypertexte, qui permettent de naviguer entre les pages web, et il est utilisé pour programmer de manière déclarative, en décrivant la structure et le contenu de la page plutôt qu'en spécifiant des actions spécifiques.

  3. Page web et site web :
    4.

        Une page web est un document multimédia contenant du texte, des images, des liens et des possibilités d'interaction. Elle est décrite élément par élément à l'aide de langages de description tels que HTML, XHTML ou HTML5, et est stockée dans un fichier HTML. Une page web est interprétée et affichée par un navigateur, et elle est localisée sur Internet grâce à une adresse URL. Une page web permet également d'accéder à d'autres pages en suivant des liens. Un site web, quant à lui, est un regroupement de pages web liées entre elles par des liens hypertextes.Il est généralement émis par une entité spécifique et est accessible à partir d'une page d'accueil et d'une URL de base communes. Les sites web sont souvent organisés en arborescence de dossiers et de fichiers, et permettent de définir des chemins relatifs.

  4. Création d'une page web HTML :
    5.

        Un document HTML est un fichier texte structuré à l'aide de balises. Il est interprété et visualisé dans un navigateur. La création d'une page web HTML peut se faire à l'aide d'un éditeur de pages web tel que Dreamweaver, ou simplement avec un éditeur de texte tel que Bloc-notes ou Wordpad.
    Les principes du langage HTML incluent l'utilisation de balises telles que <html>,<head>,<title>,<body> et pour délimiter le début, l'entête, le titre et le corps du document. Des balises spécifiques comme <h1> à <h6> sont utilisées pour les titres de niveau correspondant, et <p> est utilisé pour les paragraphes.
        En résumé, le Web est un système hypertexte public sur Internet qui permet d'accéder à des documents textuels reliés entre eux par des liens hypertexte.
    Les URL sont utilisées pour localiser les ressources sur des serveurs accessibles via Internet. HTTP facilite les communications entre les navigateurs et les serveurs web, et HTML est utilisé pour structurer et présenter les pages web de manière interactive.

  1. Réseaux Network :
    2.

        Est un ensemble d'ordinateurs de périphérique autonomes connectés entre eux pour échanger des informations numériques selon des règles bien définies.
    Mise en réseaux :
        est la mise en œuvre d'outils et de taches pour relier les ordinateurs afin qu'ils puissent partager des ressources.
    Evolution :
        les réseaux informatiques sont nés du besoin de faire communiquer des terminaux distants avec un site centrale puis des ordinateurs entre eux. Un réseau repose sur deux notions fondamentales :
    • L'interconnexion.
    • La communication.

  2. Les caractéristiques de base d'un réseau :
    3.

    • La topologie( physique, logique ).
    • Le débit exprimé en bits/s (ou bps).
    • La distance maximale (ou portée).
    • Le nombre de nœuds maximum.

  3. Quelques terminologies en réseau :
    4.

    • Station de travail (PC, MAC, SUN Terminal X, …).
    • Nœud : C'est une station de travail, une imprimante, un serveur ou toute entité pouvant être adressée par un numéro unique.
    • Paquet : C'est la plus petite unité d'information pouvant être envoyé sur le réseau.
    • Réseaux Hétérogènes : Les ordinateurs reliés au réseau sont de constructeurs divers.
    • Le Débit mesure une quantité de données numériques (bits).

  4. Éléments d'un réseau :
    5.

    • Les ordinateurs équipés d'une carte de communication.
    • Les logiciels navigateur, client de messagerie, serveur web, …
    • Les supports du : LAN, prises RJ45, WIFI: WAN l, ADSL, fibre optique, …
    • Les équipements d'interconnexion du LAN : concentrateur, commutateur, WAN...

  5. Classification des réseaux :
    6.  5.1. Par portée l'envergure (La distance) :
    LAN, MAN et WAN.
    • LAN (Local Area Network = réseau local d'entreprise) :

        Est constitué d'ordinateurs et de périphériques reliés entre euxdans une même entreprise, ce type de réseaux considéré commeprivé.( centaines de machines, vitesse de transmission entre 10 à 100 Mb/s).

    • MAN : (Métropolitan Area Network = Réseau métropolitain ou urbain) :

        Correspond à la réunion de plusieurs réseaux locaux (LAN) dans une zone géographique spécifique il utilise le cablecaoxial comme support physique pour assurer une transmission de données efficace sur une longue distance.

    • WAN : (Wide Area Network = réseau grande distance) :

        Est un type de réseau informatique qui couvre de vaste distance, que ce soit à l'échelle d'un pays ou à l'échelle mondiale. Il est composé de réseaux publics.

    5.2. L'ouverture (leur utilisation) :

    • Intranet (privés internes).
    • Extranet (privé internes et externes).
    • Internet Les réseaux publics (réseau des réseaux), nationaux ou internationaux interconnectés à l'échelle de la planète.

    5.3. Les supports de connexion :

    • Réseaux filaires : (Réseau Téléphonique Commuté)
    • Réseaux sans fils : Les ondes radios, Les satellites, Le Bluetooth ….

    5.4. Le type d'organisation :

    • Les réseaux Postes à Postes : Chaque ordinateur est un peu serveur et un peu client. Chaque ordinateur est libre de partager ses ressources.
    • Les réseaux Clients/Serveurs: La plupart des stations sont des « postes clients »

  6. le type de topologie :
    7.  Les réseaux en bus:

        Les machines sont reliées par un câble coaxial (le bus), et chaque ordinateur est connecté en série sur le bus par l'intermédiaire de connecteurs BNC. Cependant,ces types de réseaux ont perdu en popularité ces dernières années en raison de leurs limitations.

    Les réseaux en étoile:

        Toutes les stations sont connectées à un commutateur. SWITCH(HUB = concentrateur) : Les stations émettent vers ce concentrateur qui renvoie les données vers tous les autres ports réseaux (hub) ou au destinataire (Switch).

    Les réseaux en anneau :

        Les ordinateurs sont situés sur une boucle et communiquent chacun à leur tour. Elle utilise la méthode d'accès à « jeton ». Les données transitent de stations en stations en suivant l'anneau qui chaque fois régénèrent le signal.

    Maillée:

        Internet est une topologie maillée sur le réseau étendu « WAN ». Les réseaux maillés utilisent plusieurs chemins de transferts. C'est une structure réseau hybride reprenant un câblage en étoile regroupant différents nœuds de réseaux.


Chapitre III: Les modèle OSI

  • NAKIB Zahra/202031028292/MI/Section3/Groupe2


  1. Les composants matériel d'un réseau informatique:
    2.  1-Les cartes réseau(NIC):

        Sont des composants installées à l'intérieur de l'ordinateur pour lui permettre de se connecter à un réseau tel qu'internet ou bien un réseau local , par une prise Ethernet ou WI-FI. Ces cartes sont responable de permettre à l'ordinateur de transmettre et de recevoir des donnée par des protocoles de communication spécifiques.

    2-Le Switch:

        Le commutateur réseau est utulisé pour connecter des appareils sur un réseu local et permet la communication entre eux en utilisant leurs adresses MAC stocker dans une table de correspondance à l'interieure du Switch,où l'orsqu'un appareil souhaite communiquer avec un autre le switch utilise cette table pour envoyer les données à l'appareil destinataire uniquement. Les commutateur réseau sont disponibles en différentes tailles, petits pour les réseaux domestiques et des Switch de niveau entreprise pour des plus grands réseaux.

    3-HUB:

        Le concentrateur est utilisé pour connecter plusieurs appareil sur un réseau local(LAN).Le hub reçoit les données envoyées par un appareil et les transmet à tous les appareils connectés contrairement à un switch qui les envoient seulement à l'appareil destinataire , et c'est pour ça les hubs ont été largement remplacés par des switch pour une communication plus rapide et sécurisée entre les appareils connectés(les hubs sont encore utilisés dans des réseau de petite taille).

  2. Le modéle OSI:
    3.  -Définition:

        OSI est un modèle de référence pour les communications de réseau , il est souvent utilisé comme référence pour la conception et la mise en oeuvre de réseaux informatiques. Ce modéle consiste à diviser la communication entre les appareils de réseau en sept couches :

      • La couche physique: Responsable de la transmission des données sur le support physique.
      • La couche liasion de données: Responsable de la transmission des données entre les appareils qui ont le même support physique.
      • La couche réseau : Responsable du routage des données à travers le réseau.
      • La couche transport : Responsable de l'acheminement des données de bout en bout entre les appareils.
      • La couche session : Responsable de l'établissement et de la gestion des sessions entre les appareils.
      • La couche présentation : Responsable de la conversion des données dans un format qui peut être compris par l'appareil de destination.
      • La couche application : Responsable de la communication entre les applications sur les appareils de réseau.

    -Remarque:

        Le modéle OSI est un modéle de référence théorique et n'est pas utilisé dans les protocoles de communication réseau moderne mais il reste utile pour comprendre les principes fondamenteaux de la communication réseau.

  3. Le modèle TCP/IP:
    4.  -Définition:

        Le modèle TCP/IP est un modèle de référence pour les communications de réseau, qui est utilisé pour la conception et la mise en œuvre des réseaux informatiques modernes.Le modèle TCP/IP divise les communications en 4 couches principal:
      • La couche liaison de données: cette couche est responsable de la transmission des données sur le support physique.
      • La couche réseau: : cette couche est responsable du routage des données à travers le réseau.
      • La couche transport: cette couche est responsable de l'acheminement des données de bout en bout entre les appareils.
      • La couche application: cette couche est responsable de la communication entre les applications sur les appareils de réseau.

  4. L'adressage:
    5.  -Les communications dans un réseau obéissent à des règles :

      • Adressage: Permet d'identifier de manière unique les deux unités en communication.
      • Architecture: Définit les rôles endossés par les deux unités.
      • Protocoles: Assurent l'échange des données.

    -L'adressage IP :

        Une adresse IP est le numéro qui identifie chaque ordinateur dans les réseaux qu'il appartient. Il y'a deux mode d'adressage IPV4 et IPV6 :

    1. ADRESSE IPV4:

       a. Chaque paquet contient : adresse source sur 32 bits,et une adresse destinataire sur 32 bits.
       b. Donc IPV4 == 4 octets == 32bits, représenter par 4 chiffres en décimale séparé par des points(chaque chifre regroupe un octes).

    2. ADRESSE IPV6 :

       a. Les adresses sont codées sur 128 bits. La représentation textuelle se fait en découpant le mot de 128 bits de l'adresse en 8 mots de 16 bits séparés par le caractère «:», chacun d'eux est en hexadécimal.

  5. L'adresse IPV4:
    6.

    • Chaque adresse IPV4 contient deux parties:

      o Une première partie des bits: Adresse réseau.
      o Une deuxième parties des bits: Adresse hôte.
    • On reconnait qu'un réseau est un {hôte} quand la valeur des bits pour la partie réseau est lamême.
    • Le nombre des bits contenus dans la partie hôtes détermine le nombre des machines possible sur ce réseau.
    •l'adressage IPV4 s'effectué selon les types suivants :

    1) L'adresse réseau:

    Tous les bits de la partie hôtes=0,c'est-à-dire tous les hôtes(machines) de ce réseau ont la même partie réseau.

    2) L'adresse de diffusion:

    il s'agit d'une adresse dans laquelle tous les bits de partie hôtes=1,pour le réseau. Cette adresse permette à un pc d'envoyer des paquets à l'adresse de diffusion de réseau, et tous les hôtes de cette adresse reçoivent ces données, sans qu'il distribue un paquet pour chaque pc d'une manière unique.

    3) L'adresse hôtes :

    On attribue les valeurs situés entre l'adresse réseau et l'adresse de diffusion sur machines situé sur cette réseau.

    4) Le masque de réseau et de sous réseaux :

    Tous les bits de la partie réseaux = 1.

Chapitre V: Système de Gestion de Version

  1. Qu'est ce que Git:
    2.

        Git est un logiciel de gestion de version créé par Linus Torvalds en 2005 pour faciliter la collaboration et le suivi des modifications dans les projets de développement de logiciels. En d'autres termes, Git permet à plusieurs personnes de travailler sur un même projet et de suivre les modifications apportées aux fichiers au fil du temps.

  2. Les modèles de gestions de versions:
    3.

        Il y a 2 logiciels de gestion de version : le modèle centralisé et le modèle décentralisé. Dans un modèle centralisé, le code source du projet est hébergé sur un serveur central distant, tandis que dans un modèle décentralisé, chaque utilisateur télécharge et héberge le code source sur sa propre machine. Le modèle décentralisé est présenté comme étant plus simple et flexible pour les utilisateurs, ainsi que plus sécurisé, car chaque utilisateur possède une copie complète du code en cas de corruption du serveur central.

  3. Github:
    4.

        GitHub est un service en ligne qui héberge des dépôts Git. Les dépôts sont des copies de l'intégralité des fichiers d'un projet et de leurs versions situées sur un serveur central. GitHub permet de contribuer à des projets en proposant de nouvelles fonctionnalités. Git est un logiciel de gestion de version utilisé avec GitHub.
        Un "dépôt" correspond à l'ensemble des fichiers d'un projet importé dans Git et qu'il y a deux façons de créer un dépôt Git : importer un répertoire existant ou cloner un dépôt Git existant.
        Dans Git, les données sont stockées sous forme d'instantanés ou "snapshots". Lorsqu'un utilisateur enregistre l'état d'un projet dans Git, Git prend une capture de l'état du contenu de l'espace de travail à ce moment-là et enregistre une référence à cet instantané pour un accès ultérieur. Chaque instantané est stocké dans une base de données locale. Étant donné que Git dispose de l'historique complet d'un projet localement, la majorité des opérations de Git peuvent être réalisées localement, ce qui rend le travail plus rapide et plus agréable.
        Dans Git, un fichier peut être soit “suivi” soit non “suivi” . Les fichiers sont considérés comme “suivis” s'ils appartiennent au dernier instantané capturé par Git et donc stockés dans la base de données locale. Chaque fichier suivi peut avoir trois états: modifié, indexé et validé. Un fichier modifié est considéré comme modifié par Git tant qu'il n'a pas été indexé. On indexe un fichier lorsqu'on lui indique qu'il doit faire partie du prochain instantané dans sa version actuelle. Enfin, lorsqu'on demande à Git de prendre l'instantané, les fichiers faisant partie de l'instantané sont considérés comme validés et le cycle recommence. Un fichier “non suivi” en revanche, est un fichier qui n'est pas inclus dans le dernier instantané capturé par Git et n'est donc pas stocké dans la base de données locale.

  4. Gérer un dépôt Git:
    5.

        Il est possible d'importer toutes les ressources d'un projet existant dans Git en utilisant la commande "git init" pour initialiser un dépôt Git. Pour indexer des fichiers, on utilise la commande "git add", puis on utilise la commande "git commit" pour valider ces fichiers et les ajouter en base.
        La deuxième façon de démarrer un dépôt Git est de cloner localement un dépôt Git déjà existant en utilisant la commande "git clone". Lorsqu'on travaille sur un projet avec Git, on doit utiliser les commandes "git add" et "git commit" pour enregistrer les modifications apportées aux fichiers dans le dépôt Git. La commande "git log" permet de consulter l'historique des modifications Git.
        En utilisant la commande "git checkout" ou "git restore", il est possible d'annuler les modifications apportées à un fichier si les résultats ne sont pas ceux escomptés. Enfin, une branche en Git est simplement un pointeur vers un commit spécifique qui permet de créer de nouvelles branches pour tester des fonctionnalités sans affecter la branche principale.
